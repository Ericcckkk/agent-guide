# Agent Collaboration Guide

这是给 AI Agent 的协作手册。每次新对话开始时，请先阅读本仓库相关文件，了解我的工作偏好和历史踩坑经验。

## 使用方式

对话开头告诉 agent：
> 先读一下 https://github.com/Ericcckkk/agent-guide 里的相关文件，了解我的偏好和规则。

## 文件结构

| 文件 | 内容 |
|------|------|
| `general.md` | 通用协作原则（适用所有项目） |
| `frontend.md` | 前端开发相关规则 |
| `deployment.md` | 部署/CI-CD/运维相关 |
| `server-infra.md` | 我的服务器环境信息 |
| `coding-style.md` | 代码风格偏好 |
| `projects.md` | 项目协作协议（跨设备接手项目的标准流程） |
| `troubleshooting.md` | 踩坑速查索引（按症状查解决方案） |
| `emergency.md` | 应急与回滚（危险操作清单 + 恢复方法） |

## 贡献规则（给 Agent 的）

当我让你更新这个仓库时，必须遵守以下规则：

### 只追加，不覆盖

- **只能在现有文件末尾追加新条目，不能修改或删除已有条目。**
- 如果觉得某条旧规则过时了，在它下方加一条标注 `[更新 YYYY-MM-DD]` 的新条目说明变化，不要直接改原文。
- 新增条目格式统一：`- [ ] **粗体摘要。** 详细说明。`

### 新增文件

- 如果经验属于全新领域（比如移动端开发、数据库设计），可以新建 `.md` 文件。
- 新文件必须在 README 的文件结构表格中注册。
- 文件名用英文小写 + 连字符（如 `mobile-dev.md`）。

### 质量门槛

- **只记录经过验证的经验。** 不要把猜测或未验证的建议写进来。
- **必须是可复用的通用规则。** 一次性的、只对当前对话有用的信息不要写。
- **每条规则必须有明确的行动指引。** 不要写"注意 xxx"，要写"做 xxx / 不要做 xxx"。
- **不要重复。** 新增前先检查是否已有类似条目。

### Commit 规范

- 消息格式：`docs: 简短描述`（如 `docs: add backend error handling rules`）
- 一次 commit 只改一个主题，不要混杂多个不相关的修改。

## 何时触发更新

Agent 在以下时机应主动提议更新本仓库（需经我确认）：

- **踩坑修复后** — 某个方案失败、换了方案才成功，记录"不要做 X，要做 Y"。
- **我明确表达偏好时** — 我说了"以后都这样做"、"我不喜欢这种方式"、"别再 xxx 了"。
- **发现环境变化时** — 比如新开了端口、加了新项目、换了服务器配置。
- **对话结束前** — 如果本次对话中产生了符合质量门槛的新经验，主动问我："这次有几条新经验，要更新到 agent-guide 吗？"

**不要自动更新，每次都先列出要追加的条目让我确认。**

## 给我（Eric）的使用建议

新对话开头这样说：

```
先读一下 https://github.com/Ericcckkk/agent-guide 里的文件，了解我的偏好和规则，整个对话过程中严格遵守。
如果过程中踩了坑或者我表达了新的偏好，对话结束前问我要不要更新 agent-guide。
```

这样每个 agent 都会读规则、遵守规则、并且帮你持续积累经验。
