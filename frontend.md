# 前端开发规则

## 技术栈偏好

- [ ] **轻量优先。** 能用 Tailwind + Alpine.js 解决的不要上 React/Vue 全家桶。
- [ ] **CDN 引入。** 个人项目不搞 npm build，直接 CDN script 标签引入。
- [ ] **单文件优先。** 能一个 HTML 搞定的不要拆成多个文件。
- [ ] **ECharts 做图表。** 不用 Chart.js 或 D3。

## 设计风格

- [ ] **简洁高级感。** 参考 Apple 风格，大留白、圆角卡片、微阴影。
- [ ] **配色克制。** 主色用深色（#0f172a），强调色用绿色（#10b981），不要花花绿绿。
- [ ] **字体用系统字体栈。** SF Pro Display → Inter → system-ui。
- [ ] **图标用内联 SVG。** 不引入 icon 库。
- [ ] **App 图标/视觉设计要对标获奖水准。** 不要凑合，要有明确的设计理念。双色优于三色，要有粗细对比、黄金分割等设计语言。字体要有艺术感，不能用普通系统字体直出。

## 移动端

- [ ] **必须手机可用。** 所有页面都要响应式，手机上能正常操作。
- [ ] **PWA 支持。** 加 manifest.json + service worker，能添加到主屏幕。
- [ ] **触摸友好。** 按钮够大、间距够宽、表单好点击。

## 数据加载

- [ ] **API 优先，本地降级。** 先尝试从 API 获取数据，失败时 fallback 到本地 JSON。
- [ ] **不要硬编码数据。** 数据从接口或文件加载，不要写死在 HTML 里。
